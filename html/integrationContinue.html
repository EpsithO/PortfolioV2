Intégration continue

L'intégration continue se base sur plusieurs principes dans un premier temps, on récupère le code enregistré dans un répertoire Bitbucket pour l’injecter dans jenkins via la commande git clone.

Ensuite les couches du projets sont builder via des esclaves jenkins, ensuite le builds des différentes couches, une fois le code examiné pour avoir un aperçu de la qualité du code fournis dans les différentes couches.

Une fois les builds terminés, un livrable est fourni puis poussé dans le répertoire Nexus.
